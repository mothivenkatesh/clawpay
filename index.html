Architecture

Tell it what you want in plain English. It wires up Cashfree APIs for you.

AI Parser: Plug Cashfree Rails in Seconds

You describe the flow. Clawpay generates the YAML, maps it to Cashfree endpoints, wires trigger → rule → action. Done.

# You describe: "Verify Aadhaar, match face, check CIBIL > 680, disburse 80/20 co-lending split" # Clawpay generates: trigger: webhook/loan-apply steps: - verify: smartocr/aadhaar - verify: facematch - rule: cibil_score > 680 - allocate: bank_a: 80% nbfc_b: 20% - execute: cashfree/payouts

Three-Layer Architecture

Trigger Layer: Webhooks, schedules, form submissions, API events, doc uploads

Rule Engine: Eligibility, allocation, risk, compliance, workflow routing, retry & fallback. DAG-based. No-code visual builder. Maker-checker.

Action Layer (Cashfree): SmartOCR, FaceMatch, Liveness, Payment Links, Forms, Payouts, Bank Verify, PG

Why this architecture wins

Razorpay burns tokens on every action. Clawpay runs rules at zero cost.

Razorpay Agent Studio

Every action goes through Claude SDK. Observe → Reason → Execute. Works fine for 50 cart recoveries/day.

Token cost per action. Scales with usage volume. For a bank running continuous monitoring on 10M accounts, the LLM cost would be astronomical.

Clawpay

Rules are code, not LLM calls. AI only touches config generation (one-time) and edge cases (1% of decisions). The rest is free.

Zero token cost at runtime. 99% of decisions are deterministic rules. AI touches config generation + edge cases only. Scales to millions of accounts.

How it should work: AI generates the config (one-time, high value). Rules execute at zero cost (infinite scale). AI handles edge cases (1%, worth the tokens). That’s it.
